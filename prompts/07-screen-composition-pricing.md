# Prompt 07: Screen Composition — Pricing Page

## Prompt

```
Design the screen composition for a SaaS pricing page using the screen-composition-design skill.

Context: B2B analytics platform with 3 tiers (Starter, Pro, Enterprise). The page needs to clearly communicate value differences, guide users toward the Pro tier (best value), and handle annual/monthly toggle. Enterprise tier requires "Contact Sales" instead of direct signup.
```

## Expected Output Should Include

- **Screen layout structure** with visual hierarchy
- **Component breakdown**: Pricing cards, toggle, feature comparison, FAQ
- **Visual emphasis** on recommended tier (Pro)
- **Content hierarchy**: What users see first, scan path
- **Responsive behavior**: How layout adapts across breakpoints
- **Token usage** for spacing, typography, colors
- **Interaction details**: Toggle behavior, hover states, CTA differentiation

## Red Flags (Plugin Not Working If...)

- No clear visual hierarchy between tiers
- Missing responsive considerations
- No token references
- Treats all 3 tiers with equal visual weight
