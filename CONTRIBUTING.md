# Contributing to Premium UI Plugin

Thank you for your interest in contributing! This document provides guidelines for contributing to the Premium UI Plugin.

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest features
- Provide a clear description of the problem or suggestion
- Include steps to reproduce if reporting a bug

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Ensure your changes follow the existing code style and conventions
5. Commit with a clear message (`git commit -m "Add: description of change"`)
6. Push to your fork (`git push origin feature/your-feature`)
7. Open a Pull Request

### Adding or Modifying Skills

When contributing new skills or modifying existing ones:

- Follow the existing skill structure (`SKILL.md` + supporting rules file)
- Maintain the premium UI philosophy — no cheap patterns, no gimmicks
- Ensure the skill has a clear, focused purpose
- Include proper frontmatter (`name`, `description`) in `SKILL.md`
- Test the skill with Claude to verify it produces quality output

### Writing Standards

- Be clear and direct
- Avoid vague or generic language
- Follow the tone of existing documentation
- Keep instructions actionable and specific

## Code of Conduct

Please be respectful and constructive in all interactions. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for details.

## Questions?

Open an issue if you have questions about contributing.
